## 📝 Git提交规范

### 提交信息格式（Conventional Commits）

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型（type）**：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具相关

**示例**：
```bash
git commit -m "feat(algorithm): 添加知识点切分算法

- 实现基于语义相似度的知识点切分
- 支持ASR和OCR文本输入
- 添加单元测试

Closes #123"
```

## 🔄 日常开发流程

### 1. 开发新功能

```bash
# 1. 创建功能分支
git checkout -b feature/新功能名称

# 2. 开发代码
# ... 编写代码 ...

# 3. 提交代码
git add .
git commit -m "feat: 新功能描述"

# 4. 推送到远程
git push origin feature/新功能名称

# 5. 创建Pull Request（在GitHub上）
```

### 2. 修复bug

```bash
# 1. 创建bugfix分支
git checkout -b bugfix/问题描述

# 2. 修复代码
# ... 修复代码 ...

# 3. 提交修复
git commit -m "fix: 修复问题描述"

# 4. 推送到远程
git push origin bugfix/问题描述
```

### 3. 代码审查

- 团队成员提交Pull Request
- 队长审查代码
- 通过后合并到develop分支
