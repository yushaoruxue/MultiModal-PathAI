# AI赋能职教视频个性化教学项目 - 任务分配文档

## 项目基本信息
- **项目名称**: AI赋能职教视频个性化教学项目
- **项目周期**: 16周
- **团队组成**: 1名队长 + 4名队员
- **文档版本**: v1.0
- **制定日期**: 2024年

---

## 团队成员与角色

| 角色 | 姓名 | 主要职责 |
|------|------|----------|
| **队长** | 周东吴 | 项目统筹、需求对接、核心模块开发、技术攻关 |
| **队员1** | 成怡 | 前端开发（学生端+教师端界面） |
| **队员2** | 杨雅娟 | 后端开发（业务逻辑+API接口） |
| **队员3** | 张亚鹏 | AI引擎集成（多模态解析+难点识别） |
| **队员4** | 雷宇 | 数据模型设计+测试+文档 |

---

## 总体任务分配原则

1. **队长承担更多核心开发任务**，同时负责项目统筹和技术难点攻关
2. **队员按技术栈分工**，前端、后端、AI、数据测试各司其职
3. **模块化协作**，定期同步进度，确保接口对接顺畅
4. **优先级明确**，先完成MVP核心闭环，再扩展功能

## 协作接口定义

### 前端 ↔ 后端接口
- **负责人**: 成怡（前端） + 杨雅娟（后端）
- **对接时间**: 第3周开始，每周同步
- **主要接口**:
  - 行为事件上报：`POST /api/watch-events`
  - 补偿资源获取：`GET /api/knowledge-points/{kp_id}/resources`
  - 学习路径查询：`GET /api/users/{user_id}/learning-path`
  - 练习提交：`POST /api/exercises/{exercise_id}/submit`

### 后端 ↔ AI引擎接口
- **负责人**: 杨雅娟（后端） + 张亚鹏（AI引擎）
- **对接时间**: 第4周开始
- **主要接口**:
  - 视频解析请求：`POST /api/ai/parse-video`
  - 知识点切分：`POST /api/ai/segment-knowledge-points`
  - 补偿资源生成：`POST /api/ai/generate-resource`

### 算法模块 ↔ 后端接口
- **负责人**: 周东吴（算法） + 杨雅娟（后端）
- **对接时间**: 第5-7周
- **主要接口**:
  - 难点判定：`POST /api/algorithm/detect-difficulty`
  - 路径生成：`POST /api/algorithm/generate-path`
  - 路径调整：`POST /api/algorithm/adjust-path`

### 数据模型对接
- **负责人**: 雷宇（数据设计） + 杨雅娟（数据实现）
- **对接时间**: 第2-3周
- **主要工作**:
  - ER图评审
  - 表结构确认
  - 索引策略确认

## 风险应对措施

### 技术风险
- **多模态解析效果不稳定** → 队长+张亚鹏协作攻关，提供人工编辑兜底
- **难点判定误报/漏报** → 队长优化算法，雷宇加强测试验证
- **补偿资源质量参差** → 张亚鹏优化生成策略，队长设计质量评估机制

### 进度风险
- **开发进度延误** → 队长及时调整任务优先级，必要时缩减功能范围
- **接口对接不顺畅** → 提前定义接口规范，每周同步对接进度

### 协作风险
- **沟通不畅** → 建立团队群组，每日简短同步，每周例会
- **代码冲突** → 制定代码规范，使用Git分支管理，代码审查

## 注意事项

1. **代码规范**: 统一代码风格，使用Git进行版本控制，提交代码前进行自测

5. **进度透明**: 遇到问题及时在团队群组中反馈，不要拖延
